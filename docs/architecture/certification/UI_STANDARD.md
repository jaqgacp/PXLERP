# UI Standardization

Every Master Data module MUST follow a consistent structure. Any deviation is considered a bug.

## 1. List Page Structure
- **Toolbar:** `div.erp-list-toolbar`. Only active, implemented actions allowed.
- **Search:** `div.erp-list-search`. Debounced 300ms minimum.
- **Advanced Filter:** Optional placeholder `div.erp-advanced-filter`.
- **Grid:** `table.erp-list-table`. Dynamic headers generated by `ErpListHelper`.
- **Pagination:** `div.erp-list-pagination`. `.range()` driven.
- **Status Bar:** `div.erp-list-status`.

**Styling Rules:**
- No unnecessary whitespace.
- No inline styles.
- No emojis.

## 2. Toolbar Standard
Only display actions that actually work.
**Common Actions (Always allowed if implemented):**
- New
- Download Template
- Import
- Export
- Print
- Refresh

**Conditional Actions (Only if implemented):**
- Approve, Post, Generate, Reverse, Void, Lock, Submit.

**Fake Buttons are Strictly Forbidden.**
