# License Recommendation

**Status:** PENDING OWNER DECISION

The PXL ERP repository currently does not contain a `LICENSE` file.
By default, this means the code is under exclusive copyright ("All Rights Reserved"), even if it is hosted in a public repository.

## Action Required

Before public release or inviting open-source contributors, the repository owner must select and commit a `LICENSE` file to the root of the repository.

### Options to Consider:

**1. If you want to keep the software proprietary and closed-source (but visible for portfolio/audit reasons):**
- Do nothing, or add a LICENSE file explicitly stating "All Rights Reserved. No permission is granted to use, modify, or distribute this software."

**2. If you want a standard open-source license with high permissiveness (anyone can use and commercialize):**
- **MIT License:** Simple, highly permissive.
- **Apache License 2.0:** Permissive, but includes explicit patent grants and trademark protections.

**3. If you want a copyleft license (requires derivative works to be open-sourced):**
- **GNU General Public License (GPLv3):** Strong copyleft. Any modifications distributed must also be GPLv3.
- **GNU Affero General Public License (AGPLv3):** Strong copyleft designed for network software. If someone hosts a modified version of PXL ERP as a service, they must release their modified source code.

> [!CAUTION]
> **Do not invent legal terms.** Please use a standard OSI-approved license. Visit [choosealicense.com](https://choosealicense.com) for guidance, and consult legal counsel if this is a commercial enterprise product.
